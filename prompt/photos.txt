需求概述
实现一个单页相册浏览应用，所有交互均在同一 URL 下完成（无页面刷新，无路由跳转）。
页面默认展示相册列表（交错堆叠卡片，每张卡片内的多张缩略图以不同角度旋转叠加，营造手工拼贴感）。点击任意相册卡片后，列表视图被隐藏，取而代之的是该相册的详情视图，展示该相册内的全部图片。

交互逻辑
列表 → 详情切换：点击相册卡片 → 隐藏 .album-container（相册列表），显示 .detail-container（详情视图）；点击“返回画廊” → 隐藏详情，显示列表。

状态管理：使用 JavaScript 控制显示/隐藏（如 display: none/flex 或 v-show），无需改变 URL 或刷新页面。

详情视图内容
顶部左侧：“返回画廊” 文字按钮（带左箭头图标），点击返回列表。

顶部右侧：相册日期（如 2026-05-07）。

中部：相册标题（大号，如“二六年南昌五一摄影”）。

副标题/描述（小字，如“随便拍拍”）。

下方：该相册所有图片以 整齐的网格布局 展示（建议 2~4 列，响应式调整）。

每张图片：

无白色边框（取消原有的 padding 白边）。

圆角处理：棱角磨圆润（如 border-radius: 12px 或 16px），保持柔和。

不旋转：所有图片水平放置，无任何角度倾斜，整齐排列。

尺寸适中：图片高度建议控制在 150px ~ 180px（视屏幕调整），不宜过大，以保持浏览舒适度。

悬停效果：鼠标悬停时轻微放大（如 scale(1.03)）或改变阴影，增强交互反馈。

点击放大：点击任意图片可弹出 Lightbox 灯箱（全屏遮罩，居中显示大图，点击遮罩或关闭按钮可关闭），便于查看细节（非必须，但建议实现）。

UI 样式细节
整体背景：保持深蓝色渐变 + 微光氛围（与现有 body 样式一致）。

详情视图背景：与列表视图相同，无需额外容器背景。

图片缩略图：使用 object-fit: cover 统一裁剪，确保网格对齐。

标题、日期、描述：文字颜色、字体、间距与列表中的信息区保持一致。

“返回画廊”按钮：样式为无背景的文字链接，颜色为白色半透明，悬停变亮，带左箭头图标。

响应式适配
桌面端（>900px）：网格 3~4 列，间距适中。

平板（≤900px）：网格 2~3 列。

手机（≤480px）：网格 1~2 列，图片高度适当缩小（如 120px），标题字号减小。

视觉风格区分（重点）
列表视图（相册卡片）：

每张卡片内的缩略图 必须保留交错堆叠效果（不同图片旋转不同小角度，叠加摆放，营造手工感），这是画廊入口的核心视觉特征。

详情视图（单相册内图片）：

所有图片 整齐排列，无旋转，无白边，圆角，尺寸适中，突出清晰浏览。

额外要求（可选）
图片加载时显示占位背景色或渐入动画（如 fade-in）。

Lightbox 放大功能优先实现（点击图片弹出大图），增强体验。

技术实现约束
仅使用 HTML + CSS + 原生 JavaScript（无第三方框架或库）。

所有数据来源于 albums 数组（或模拟数据），点击时根据索引获取对应相册数据并渲染详情视图。

代码保持模块化（如将渲染、事件绑定、Lightbox 等拆分为独立函数），便于后续维护。
